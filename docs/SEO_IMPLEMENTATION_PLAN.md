### 5.1 URL Structure Design

#### Tier 1: Service + Location (Priority 1)
**Format:** `/{service}-in-{emirate}`

**Examples:**

### 5.2 Canonicalization Strategy

**Rule 1:** Brand pages canonical to location pages
```html
<!-- /ceramic-coating-in-dubai/ferrari -->
<link rel="canonical" href="https://wrpdetailing.ae/ceramic-coating-in-dubai">
```

**Rule 2:** Location pages self-canonical (rank independently)
```html
<!-- /ceramic-coating-in-dubai -->
<link rel="canonical" href="https://wrpdetailing.ae/ceramic-coating-in-dubai">
```

**Rule 3:** Main service page self-canonical
```html
<!-- /services/ceramic-coating -->
<link rel="canonical" href="https://wrpdetailing.ae/services/ceramic-coating">
```

**Sitemap Strategy:**
- Include Tier 1 pages (service + location) in sitemap
- NOINDEX Tier 2 pages (service + location + brand) OR canonical to Tier 1
- Tier 2 pages provide internal linking and capture long-tail searches

```typescript
interface Crumb {
  name: string;
  url: string;
}

interface Props {
  crumbs: Crumb[];
  className?: string;
}

const { crumbs, className = '' } = Astro.props;

const schema = {
  '@context': 'https://schema.org',
  '@type': 'BreadcrumbList',
  itemListElement: crumbs.map((crumb, index) => ({
    '@type': 'ListItem',
    position: index + 1,
    name: crumb.name,
    item: crumb.url,
  })),
};
---

<!-- Visual breadcrumbs -->
<nav aria-label="Breadcrumb" class={`text-sm ${className}`}>
  <ol class="flex items-center gap-2">
    {crumbs.map((crumb, index) => (
      <li class="flex items-center gap-2">
        {index > 0 && <span class="text-muted-foreground">/</span>}
        {index === crumbs.length - 1 ? (
          <span class="text-muted-foreground">{crumb.name}</span>
        ) : (
          <a href={crumb.url} class="hover:underline">{crumb.name}</a>
        )}
      </li>
    ))}
  </ol>
</nav>
```
